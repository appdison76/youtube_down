# 최종 답변

## ✅ 확인된 사실

**주식계산기도 네트워크 IP로 나오는데 잘 작동합니다!**

그렇다면:
- ✅ 네트워크 IP로 나오는 것은 **정상**입니다
- ❌ 타임아웃이 발생하는 것이 **실제 문제**입니다

## 🔍 타임아웃 원인 및 해결

### 1. ADB 포트 포워딩 확인

```powershell
# ADB 포트 포워딩 상태 확인
adb reverse --list

# 포트 포워딩 설정
adb reverse tcp:8081 tcp:8081
```

### 2. 방화벽 확인

Windows 방화벽이 포트 8081을 차단하지 않는지 확인

### 3. 네트워크 연결 확인

PC와 폰이 같은 WiFi 네트워크에 연결되어 있는지 확인

## 💡 핵심

**네트워크 IP로 나와도 정상 작동해야 합니다!**

주식계산기가 네트워크 IP로 작동한다면, 현재 프로젝트도 네트워크 IP로 작동해야 합니다.

타임아웃 문제는:
- ADB 포트 포워딩
- 방화벽
- 네트워크 접근성

이 중 하나일 가능성이 높습니다.

## 🎯 확인 방법

`npx expo run:android` 실행 후:
1. 빌드가 완료되는지 확인
2. 앱이 설치되는지 확인
3. Metro 서버가 시작되는지 확인
4. 타임아웃이 발생하는 시점 확인 (빌드 중? 연결 중?)

**주식계산기와 동일하게 작동해야 합니다!**
