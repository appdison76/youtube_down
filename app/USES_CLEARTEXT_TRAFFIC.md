# usesCleartextTraffic="true" 설명

## ✅ 이 설정의 역할

`android:usesCleartextTraffic="true"`는 **HTTP 연결을 허용**하는 설정입니다.

### 왜 필요한가?

- Android 9 (API 28) 이상에서는 기본적으로 **HTTP(cleartext) 트래픽이 차단**됩니다
- Metro 개발 서버는 기본적으로 **HTTP**를 사용합니다 (`http://192.168.x.x:8081`)
- HTTPS가 아닌 HTTP 연결을 허용하려면 이 설정이 필요합니다

### 주식계산기와의 관계

- 주식계산기도 동일한 설정을 가지고 있어야 합니다
- 이 설정이 없으면 Metro 서버에 연결할 수 없습니다
- **이 설정은 필수이며, 타임아웃의 원인은 아닙니다**

## 🔍 로딩 중 타임아웃 원인

빌드 완료, 앱 설치 완료 후 **로딩 중** 타임아웃이 발생한다면:

### 가능한 원인

1. **번들 크기**: 현재 프로젝트가 주식계산기보다 더 많은 의존성을 가지고 있어 번들이 클 수 있음
2. **네트워크 속도**: WiFi 속도가 느리거나 불안정
3. **Metro 서버 응답**: Metro 서버가 번들을 생성하는데 시간이 오래 걸림
4. **타임아웃 시간**: 기본 타임아웃 시간이 너무 짧음

### 해결 방법

1. **번들 크기 확인**:
   - 주식계산기와 현재 프로젝트의 의존성 비교
   - 불필요한 의존성 제거

2. **네트워크 확인**:
   - WiFi 속도 확인
   - 같은 네트워크에 연결되어 있는지 확인

3. **Metro 서버 상태 확인**:
   - Metro 서버가 정상적으로 응답하는지 확인
   - 브라우저에서 `http://192.168.x.x:8081` 접속 시도

## 💡 핵심

**`usesCleartextTraffic="true"`는 필수 설정이며, 타임아웃의 원인은 아닙니다.**

로딩 중 타임아웃은:
- 번들 크기
- 네트워크 속도
- Metro 서버 응답 시간

이 중 하나일 가능성이 높습니다.
